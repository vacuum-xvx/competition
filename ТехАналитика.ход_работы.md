## 1. Методология и инструменты

### Гибридный подход Human-in-the-Loop (HitL)

```
ИИ-инструмент                    Аналитик (Human)
─────────────────                ─────────────────────
Генерация → ─────── REVIEW ────► Верификация фактов
                                 Архитектурный контроль
                                 Отклонение/Принятие
                        │
                        ▼
                  Корректирующий
                    промпт
                        │
                        ▼
              Итоговый верифицированный
                    результат
```

**Принцип работы:** ИИ выступал инструментом первичной генерации гипотез и структурирования данных. Аналитик применял критическое мышление: проверял каждое утверждение, особенно технические факты о стеках платформ, выявлял галлюцинации и корректировал промпты.

### Используемые инструменты

| Инструмент | Версия | Применение | Ограничения |
|-----------|--------|-----------|-------------|
| ChatGPT-4o | 2024-05 | Основная генерация, анализ требований, код | Knowledge cutoff Jan 2024; галлюцинации на фактах |
| Claude 3.5 Sonnet | 20241022 | Ревью архитектуры, контр-проверка | Не всегда знает актуальные версии OSS |
| Perplexity AI | — | Верификация технических фактов в реальном времени | Требует проверки источников |
| GitHub Copilot | — | Синтетические примеры кода | Только синтаксис, не архитектура |

### Промпт 4.2 — VR/Аудио Pipeline (Whisper)

```
РОЛЬ: Backend Engineer, специализация async processing pipeline.

ЗАДАЧА: Спроектируй асинхронный pipeline для обработки VR-аудио в Open edX.

Сценарий использования:
- Сотрудник проходит VR-тренажёр (симуляция продажи IQOS)
- Записывает аудио-ответ (WAV, ~2–5 мин)
- Система: транскрибирует → оценивает по рубрике → выдаёт feedback

Технические требования:
- STT: OpenAI Whisper (self-hosted, base/medium)
- Scoring: GPT-4o с rubric-промптом
- Хранение: MinIO (S3)
- Очередь: Celery + RabbitMQ
- xAPI: emit событие после scoring

Выдай:
1. Sequence diagram (ASCII)
2. Celery-задачу (Python, полный код с обработкой ошибок и xAPI)
3. Расчёт: сколько воркеров нужно при 50 одновременных VR-сессий?
```

**Ответ ИИ:** Код сгенерирован. Расчёт воркеров: «10 воркеров достаточно».

**⚠️ Галлюцинация #4 — расчёт воркеров:**

ИИ не учёл cold-start модели Whisper и время транскрипции:
- Whisper base: ~0.5× realtime → 3 мин аудио ≈ 90 сек CPU
- 50 сессий ÷ 90 сек × safety factor 2 = **~12–15 воркеров**

**Пересчёт аналитика:**

```
Входные данные:
  - 50 concurrent VR submissions
  - Audio duration: avg 3 min = 180 sec
  - Whisper base throughput: ~0.5x realtime → 360 sec/audio на CPU
  - GPU (T4): ~10x realtime → 18 sec/audio

CPU-воркеры: 50 ÷ (3600/360) = 50 ÷ 10 = 5 воркеров (CPU)
GPU-воркеры: 50 ÷ (3600/18) = 50 ÷ 200 = 1 GPU-воркер (с queue)

Вывод: 1 GPU-нода (T4/A10) или 8-12 CPU-воркеров для пика 50 VR-сессий.
ИИ занизил CPU и переоценил throughput.
```

**✅ Исправлено, задокументировано в архитектуре.**

Проверь расчёт стоимости GPT-4o API:
